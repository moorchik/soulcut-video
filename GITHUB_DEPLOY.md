# Як викласти сайт на безкоштовний хостинг GitHub Pages

Цей сайт можна опублікувати в інтернеті безкоштовно через **GitHub Pages**. Нижче — покрокова інструкція.

---

## Крок 1. Встановити Git (обовʼязково, якщо команда `git` не працює)

1. Завантажте установник для Windows: **https://git-scm.com/download/win**  
2. Запустіть завантажений файл і встановіть Git (достатньо стандартних налаштувань — просто натискайте Next).  
3. **Важливо:** закрийте термінал у Cursor і відкрийте його знову (або перезапустіть Cursor), щоб система підхопила команду `git`.  
4. Перевірте: у терміналі введіть `git --version`. Має зʼявитися щось на кшталт `git version 2.x.x`.

---

## Крок 2. Створити репозиторій на GitHub

1. Увійдіть на https://github.com у свій акаунт.  
2. Натисніть зелений **New** (або **Create repository**).  
3. Заповніть:
   - **Repository name:** наприклад `soulcut-video` (або будь-яка назва латиницею).
   - **Public.**  
   - **НЕ** ставлячи галочки "Add a README" / "Add .gitignore" — репозиторій має бути **порожнім**.  
4. Натисніть **Create repository**.  
5. Скопіюйте URL репозиторія — він виглядатиме так:  
   `https://github.com/ВАШ_ЛОГІН/soulcut-video.git`

---

## Крок 3. Завантажити проєкт у репозиторій

**Один раз налаштуйте імʼя та email для Git** (потрібно для комітів). У Git Bash виконайте, підставивши свої дані; email краще той, що привʼязаний до GitHub:

```bash
git config --global user.name "Ваше Ім'я"
git config --global user.email "your@email.com"
```

Далі у папці проєкту виконайте команди по черзі. Замініть `ВАШ_ЛОГІН` та `soulcut-video` на свої значення:

```bash
git init
git add .
git commit -m "Initial commit: Soulcut Video website"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГІН/soulcut-video.git
git push -u origin main
```

При першому `git push` можуть попросити увійти в GitHub (через браузер або логін/пароль). Якщо у вас увімкнена **двофакторна автентифікація**, замість пароля потрібно використати **Personal Access Token**:  
GitHub → Settings → Developer settings → Personal access tokens → Generate new token (з правами `repo`).

---

## Крок 4. Увімкнути GitHub Pages

1. У репозиторії на GitHub відкрийте **Settings** (Налаштування).  
2. Зліва виберіть **Pages**.  
3. У блоці **Source** (Джерело) виберіть:
   - **Branch:** `main`  
   - **Folder:** `/ (root)`  
4. Натисніть **Save**.

Через 1–2 хвилини сайт буде доступний за адресою:

**https://ВАШ_ЛОГІН.github.io/soulcut-video/**

(замість `soulcut-video` — назва вашого репозиторія).

---

## Custom domain (власний домен)

Якщо ви купили домен (наприклад `soulcut.video`) у реєстратора (GoDaddy, Namecheap, nic.ua, Hostpro тощо), його можна привʼязати до GitHub Pages. Зони на кшталт `.video`, `.com`, `.ua` налаштовуються однаково.

### 1. Додати домен у GitHub

1. У репозиторії: **Settings** → **Pages**.
2. У полі **Custom domain** введіть ваш домен, наприклад `www.soulcut.video` або `soulcut.video`.
3. Натисніть **Save**.
4. Якщо зʼявиться попередження про сертифікат SSL — почекайте кілька хвилин; GitHub видасть безкоштовний HTTPS.
5. За бажанням увімкніть **Enforce HTTPS**, щоб сайт відкривався тільки по https.

### 2. Налаштувати DNS у реєстратора домену

Увійдіть у панель того сервісу, де купували домен, знайдіть розділ **DNS**, **DNS-записи** або **Керування доменом** і додайте записи за одним із варіантів нижче. (Імена полів можуть відрізнятися залежно від реєстратора.)

**Варіант A — сайт відкриватиметься по `www.soulcut.video`**

| Тип   | Ім'я (Host) | Значення (Value / Points to)   |
|-------|-------------|----------------------------------|
| CNAME | www         | moorchik.github.io              |

(Замість `moorchik` — ваш логін на GitHub.)

**Варіант B — сайт відкриватиметься по `soulcut.video` (без www)**

Додайте **чотири** A-записи (для кореневого домену ім'я часто залишають порожнім або ставлять `@`):

| Тип | Ім'я (Host) | Значення (Value)   |
|-----|-------------|---------------------|
| A   | @           | 185.199.108.153     |
| A   | @           | 185.199.109.153     |
| A   | @           | 185.199.110.153     |
| A   | @           | 185.199.111.153     |

**Варіант C — працюють і `soulcut.video`, і `www.soulcut.video`**

Зробіть і варіант A (CNAME для `www`), і варіант B (чотири A-записи для `@`). У GitHub у **Custom domain** можна вказати той адрес, якою ви хочете користуватися за замовчуванням (наприклад `soulcut.video`).

Після збереження DNS оновлення можуть зайняти від кількох хвилин до 24–48 годин. Перевірити можна, відкривши ваш домен у браузері.

---

## Оновлення сайту пізніше

Після змін у файлах виконайте в папці проєкту:

```bash
git add .
git commit -m "Опис змін"
git push
```

Сайт на GitHub Pages оновиться автоматично за кілька хвилин.
