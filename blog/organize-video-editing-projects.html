Mastering Project File: A Guide for Wedding Videographers and Editors

Mar 1, 2025 · 5 min read

Opening project files should be a seamless process, but it often isn’t. Compatibility issues, missing media, and software glitches can disrupt your workflow. Here’s a clear system for organizing files so collaboration between videographers and editors stays smooth.

When many editors and clients work together, a consistent folder structure is essential. Everyone works from the same layout, which reduces mix-ups and speeds up handoffs. Below is a structure that works well for wedding film projects.

Folder structure

When you receive a project, you typically get access to a main project folder (e.g. on Dropbox, Google Drive, or WeTransfer). Inside it, a dedicated !Project folder keeps everything in one place. Open it and you’ll usually see three main folders:

Project

This folder contains the project file itself (e.g. Premiere Pro, DaVinci Resolve).

Sounds

All audio lives here, often in subfolders such as:

  • Audio Audition — processed dialogue or sound from Adobe Audition (or similar)
  • Music Film — music for the full film
  • Music Highlight — music for the highlight
  • Music Teasers — music for teasers
  • SFX — sound effects

Other audio subfolders can be added depending on the project.

Add Files

Extra assets the editor uses: LUTs, light leaks, transitions, titles, fonts, and similar. Kept separate so they’re easy to find and don’t clutter the main media.

How to open the project

Download the shared folder first, then open the project in your NLE (we’ll use Premiere Pro as an example). Go to !Project → Project and open the project file. Names often include a project number and the couple’s names in brackets.

When you double‑click the project, the software may look for media and show a Link Media window with “Missing media for these clips”. That usually means paths differ between your machine and the editor’s (e.g. different drive letters or macOS vs Windows).

Relinking is straightforward:

  • Select one missing clip in the list and click Locate (or equivalent).
  • Point the software to the same file on your computer. If you’re not sure where it is, check the “Last Path” line in the window or use the Search option so the program can find it on your drive.
  • Confirm. The window may appear again for other clips; repeat until everything is linked.
  • When all media are linked, save the project. Give Premiere Pro a few minutes (5–10) to build cache before you start editing.

Inside the project

Once the project is open, you’ll see a structure along these lines:

  1. Footage — source files you sent (we don’t rename or move client files; they stay as you sent them).
  2. Sounds — same as the Sounds folder in the file explorer (music, SFX, etc.).
  3. Multi — multi‑camera sequences (e.g. ceremony, reception, formal dances).
  4. Nest — nested sequences (e.g. for stabilisation or grouped clips).
  5. Add files — LUTs, transitions, titles, fonts, and other extras.
  6. Sequences — usually two subfolders: Films (highlights, full film, teasers) and Wedding (other timelines used during the edit).

Important nuances

1. Paths and systems. We don’t change your file or folder names. If the editor works on a different OS (e.g. Windows vs macOS), paths will differ, so relinking on your side is normal. If you receive only an XML (or similar) timeline and add it to your own project, paths can stay as on your machine.

2. Proxies. If you send 4K originals, the editor may create proxies. When you open the project, if the NLE doesn’t find those proxies on your system, you can skip relinking them; the project will still work with the originals. Proxies can be shared separately if you need them.

3. Proxy‑only delivery. If you send only proxies for editing, we treat them like originals and keep the same structure; you’ll get the project back in the same way.

4. Extra files (LUTs, transitions, etc.). When you send LUTs, light leaks, transitions, titles, or fonts, we keep them with your footage and don’t move them into the “Add files” folder in the project — they stay where you put them in the shared folder.

5. Check before you start. Before editing, confirm that all required files are present on your PC or Mac so nothing is missing mid‑project.

Final thoughts

Handling project files and relinking media well is key to smooth collaboration between videographers and editors. A shared folder structure and a few simple rules reduce mix‑ups and stress so you can focus on the cut.
